how it works.

jobscript makes it easier to run your app.

anything can go into a jobscript. this could be a clever bash script, or a command to start your program. run it on linux or in a docker container.

jobscript will run dependencies. but only when needed.

we automatically run prerequisite code. this saves a lot of headaches and prevents bugs. we also track what has been run before to save time and money.

jobscript allows you to configure how your app runs.

every job supports parameterization and overloading. this makes it easier to manage lots of configurations. and yes, jobscript will still only run these scripts when needed.

jobscript will keep your files safe.

avoid file conflicts by archiving your data to the jobscript vault. then reuse these later in other jobs.
